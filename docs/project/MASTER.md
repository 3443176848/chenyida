# 晨亿达ERP项目总控

> 最高优先级：任何新的 Codex 对话必须先阅读本文件，再继续当前任务。

## 项目名称

晨亿达ERP（Chenyida ERP）

## 项目目标

打造适用于 PCB、FPC、SMT 行业的 ERP，以统一内部编码连接物料、产品、BOM、采购、库存、生产、销售、财务和品质。

长期目标是建立 AI 物料主数据中心（Material Master），最终实现：

- AI 物料治理
- AI 采购
- AI 报价
- AI 生产辅助
- 行业物料知识库

AI 只提供建议、证据和辅助决策，不得未经审核直接创建、合并或覆盖正式物料数据。

## 当前风险

1. `chenyida_erp_site` 在根仓库中是无 `.gitmodules` 的 gitlink，新克隆无法恢复完整 Site 源码。
2. 根仓库指向 `9f2c2dc`，当前生产 Site 为 `v3` / `2b4f178`，源码归属和生产基线需要整理。
3. 本地 SQLite 与在线 D1 存在两套数据模型和两套物料编号行为，尚未确认唯一权威源。
4. 在线业务数据大量保存在 `erp_records.data_json`，关系约束、查询能力和迁移能力有限。
5. 本地数据库缺少外键和正式迁移历史；历史默认账号、弱口令和备份完整性存在风险。
6. 在线导入当前把导入行直接归为新物料，没有执行供应商映射或候选匹配。
7. 在线备份仍位于同一 D1 故障域，不能替代外部灾备。
8. 部分测试可能写入数据或受本地构建缓存影响，隔离与清理机制尚未完成。
