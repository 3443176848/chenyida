# PHASE0-TASK02 环境隔离设计

## 范围

本任务只建立开发、测试、生产环境识别和测试安全基线，不修改 ERP 业务功能、物料规则、数据库业务结构或生产资源。

## 已确认方案

- 使用根目录 `config/environments.json` 作为 development、test、production 的统一非敏感配置清单。
- Site API 烟测只允许 `ERP_ENV=test` 且目标为本机回环地址。
- Site 测试由编排脚本创建一次性 Miniflare D1 持久化目录；成功或失败后都删除整个临时目录。
- 测试失败只把去敏后的运行日志写入被 Git 忽略的 `chenyida_erp_site/work/test-logs/`，不复制 D1 文件或业务正文。
- 测试数据使用 `TEST-` 前缀，最终清理由销毁一次性数据库完成，不增加生产清理接口。
- 本地 SQLite 测试使用系统临时目录，并通过可配置的数据目录隔离备份文件；不改变备份创建和恢复函数本身。
- 远程 Test D1 只记录未来受控创建流程，本任务不创建、不连接任何云端 D1。

## 安全边界

测试守卫采用默认拒绝：环境不是 `test`、URL 不是 HTTP 回环地址、D1 路径不在系统临时目录，任一条件不满足即在网络请求或数据库创建前退出。Vite Cloudflare 插件显式关闭远程绑定，避免本地开发或测试自动使用云端资源。

凭证检查只扫描 Git 跟踪内容，拒绝提交 `.env`、私钥、数据库、备份、日志和常见真实令牌格式；`.env.example` 只能包含占位值。`.openai/hosting.json` 只允许非敏感托管元数据键。

## 验收

- 环境守卫单元测试覆盖 production、公开 URL、非临时 D1 路径的拒绝和安全 test 配置的放行。
- Site API 烟测在一次性本地 D1 上通过，测试后无 D1 业务数据残留。
- 本地 SQLite 自测、烟测、上线检查和备份恢复验证均使用临时数据库或只读/无备份模式。
- Site lint、测试、构建和凭证检查通过；生产 URL、生产 D1 和部署均未访问。
