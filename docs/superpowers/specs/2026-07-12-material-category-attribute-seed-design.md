# PHASE1-TASK03 物料分类与属性模板 Seed 设计

## 目标与范围

为在线 Site/D1 的 Material Master V2 建立第一版 PCB、FPC、SMT 行业四级物料分类、可复用属性定义和叶子分类属性模板。本任务只提供版本化 seed、隔离环境执行器、测试和治理文档，不接入 AI、Excel 导入、真实物料迁移、BOM、采购、库存或生产流程。

## 已确认决策

- 所有属性只显式绑定四级叶子分类。
- 相同模板通过声明式配置复制产生独立绑定，不实现父分类到子分类的继承、覆盖或运行时传播。
- 数据关系保持 `category -> attributes`，物料将来只选择最具体适用分类。
- seed 仅允许在 `test` 或 `local` 环境运行，禁止生产 D1 和远程绑定。

## 数据组织

新增版本化 TypeScript seed 模块，包含三个只读数据集合：

1. 四级分类节点：稳定 `code`、中英文名称、父 code、级别、启用状态、排序和说明。
2. 属性定义：稳定 `code`、名称、数据库类型、精度、标准单位、枚举值和规范化规则。
3. 叶子模板：叶子分类 code 对应显式属性列表，逐项声明必填、检索、唯一键组成和排序。

五个一级分类为 PCB/FPC 材料、电子元件、SMT 辅料、生产耗材和化学材料。所有路径补齐到四级；没有进一步产品差异的三级分类使用名称明确的“通用”或具体形态作为四级叶子，避免把物料挂在非叶子节点。

用户要求的 `NUMBER` 在既有 schema 中映射为 `INTEGER`；其余使用 `TEXT`、`DECIMAL`、`BOOLEAN`、`ENUM`。本任务不修改已执行的 `0001` migration。单位使用稳定文本代码，首版至少覆盖 `mm`、`um`、`ohm`、`%`、`W`、`kg`。

## Seed 执行器

执行器按固定顺序处理分类、属性定义和分类绑定，并在单一数据库事务中完成：

1. 在任何数据库操作前校验运行环境和全部声明式数据。
2. 按分类 code upsert，并在写入子节点前解析父分类 ID。
3. 按属性 code upsert 属性定义。
4. 按 `(category_id, attribute_definition_id)` upsert 叶子绑定。
5. 输出分类、属性和绑定各自的插入数与更新数，以及 seed 版本。

重复执行不会产生重复数据。seed 只管理其声明的稳定字段，不删除未知业务数据。任一步失败时事务整体回滚。

## 校验与错误处理

执行前必须拒绝：

- `ERP_ENV` 不是 `test` 或 `local`；
- 配置包含远程或生产数据库意图；
- 分类或属性 code 为空、重复或不符合大写字母数字加下划线规范；
- 父分类不存在、父子级别不连续、层级不是 1 至 4；
- 属性类型、单位、枚举定义不合法；
- 模板绑定到非四级叶子、未知分类、未知属性或重复属性；
- 任一四级叶子没有属性模板。

错误应指出 seed 版本和具体 code，但不输出 SQL、凭证或连接信息。

## 测试设计

使用临时目录和本机一次性 Miniflare D1：

- 应用既有 `0000`、`0001` migration 后执行 seed；
- 验证五棵分类树、四级深度、父子级差和稳定排序；
- 验证所有叶子均有属性，关键 FR4、Coverlay、电阻和锡膏模板必填项正确；
- 连续执行两次，验证第二次无新增且三类记录总数不变；
- 对空 code、重复 code、非法父子关系和非法绑定运行纯校验测试；
- 验证生产/远程环境保护；
- 继续运行原有 migration、lint、单元测试和隔离 API smoke，确认 migration 与旧业务不受影响。
