# 物料主数据中心 V2 业务决策清单

日期：2026-07-11

用途：列出需要晨亿达业务负责人明确确认的规则。未确认的阻断项不得由开发人员自行猜测后写入生产逻辑。

## 1. 阻断阶段 1 的决策

| 编号 | 需要确认的问题 | 推荐方案 | 不确认的影响 |
| --- | --- | --- | --- |
| B01 | 哪个系统是物料主数据唯一权威源 | 在线 Site/D1 为唯一生产主系统；本地 Python/SQLite 转为只读参照和迁移来源 | 两套系统继续生成不同物料和编码 |
| B02 | 正式内部物料编码格式 | 统一使用 `CYD-品类-6位流水号`，保留在线 `MAT-*` 为旧编码别名 | 新增物料继续出现编码分叉 |
| B03 | 正式编码生成时点 | 最终审核通过并启用时生成；草稿和审核中使用申请号 | 驳回申请会消耗或遗留正式编码 |
| B04 | 物料主数据业务所有者 | 指定一名数据管理员或主数据负责人，对合并、冻结、编码和规则负责 | 冲突无法裁决，开发人员被迫代替业务决策 |
| B05 | 物料生命周期 | 草稿、审核中、已启用、冻结、已停用；历史记录不删除 | BOM、采购和库存无法统一判断可用性 |
| B06 | 首期治理范围 | 高频电子料、FPC/PCB 核心材料、客户专用料和上线后新增物料优先 | 试迁移和验收范围无限扩大 |

## 2. 阻断匹配与审核的决策

| 编号 | 需要确认的问题 | 推荐方案 | 需要业务提供的内容 |
| --- | --- | --- | --- |
| B07 | 哪些字段可判定“同一物料” | 按品类定义关键属性；品牌/MPN 精确命中优先，不能只靠名称相似 | 每个首期品类的等同判定规则 |
| B08 | 自动匹配是否可以直接生效 | 只生成“自动建议”，由采购批量确认后生效；至少首期不自动写正式映射 | 可接受的人工确认工作量 |
| B09 | 自动建议和疑似匹配阈值 | 用已标注样本评估后确定，不使用拍脑袋的固定百分比 | 至少一批真实已确认正例、反例和冲突样本 |
| B10 | 冲突匹配处理责任人 | 采购提供来源，工程判断规格，数据管理员作最终合并/新建决定 | 冲突 SLA 和升级联系人 |
| B11 | 新物料审核节点 | 采购/申请人 -> 工程 -> 品质 -> 仓库 -> 数据管理员；财务仅在计价规则需要时参与 | 每个节点的必填字段和可退回原因 |
| B12 | 紧急料是否允许后补审核 | 推荐不允许跳过查重；可建立限时“临时受控”状态，但不得直接进入量产 BOM | 紧急流程适用条件、批准人和最长时限 |

## 3. 阻断下游切换的决策

| 编号 | 需要确认的问题 | 推荐方案 | 不确认的影响 |
| --- | --- | --- | --- |
| B13 | 基础库存单位和采购单位换算 | 每个物料一个基础库存单位；换算使用受控分子/分母和精度 | 库存、采购和成本无法可靠换算 |
| B14 | 客户专用料的限制范围 | 至少绑定客户，可选绑定产品；BOM、采购和领料都校验 | 客户专用料可能被其他客户项目误用 |
| B15 | 替代料分类 | 完全替代、条件替代、禁止替代；条件替代必须声明客户/产品/BOM 范围 | 生产领料无法判断替代合法性 |
| B16 | 替代料批准责任 | 工程批准技术适用性，品质批准环保/质量，客户指定时需销售或客户确认记录 | 替代关系缺少责任归属 |
| B17 | 冻结和停用对在途业务的影响 | 禁止新 BOM 和新采购；已有在途单据允许完成或由授权人关闭 | 在途采购、库存和工单会出现规则冲突 |
| B18 | 重复物料合并策略 | 选择主码，旧码转别名；历史单据不重写展示值，新业务只用主码 | 合并可能破坏历史追溯或库存对账 |

## 4. 数据迁移与验收决策

| 编号 | 需要确认的问题 | 推荐方案 | 需要人工确认 |
| --- | --- | --- | --- |
| B19 | 哪些来源进入首次迁移 | 在线 D1、当前本地 SQLite、已确认的标准物料和映射模板；其他表先只盘点 | 来源优先级和冲突时谁胜出 |
| B20 | 在线和本地同码不同数据如何处理 | 不自动覆盖，进入冲突报告，由数据管理员逐项选择或合并 | 冲突处置人和截止时间 |
| B21 | 价格、MOQ 和交期是否属于主数据 | 推荐作为供应商映射的时效属性或独立历史，不覆盖无日期的“最近值” | 价格敏感性、保留期和可见角色 |
| B22 | 历史低频物料处理 | 迁移为“待治理/冻结”并保留别名，用到时再审核启用 | 低频判定时间和用量阈值 |
| B23 | 迁移验收人 | 数据管理员核对主数据，采购核对映射，工程核对规格/BOM，仓库核对单位/库存，财务核对计价 | 每个角色的签字人 |
| B24 | 允许上线的未决冲突数 | 推荐首期范围内为 0；范围外冲突必须冻结且不得被下游新引用 | 是否接受书面例外及批准层级 |

## 5. 品类属性模板确认

首期建议至少确认以下品类。每个品类需要业务给出必填属性、可选属性、类型、单位、允许值和“同一物料”关键字段。

| 品类 | 建议首期关键字段 | 建议责任人 |
| --- | --- | --- |
| RES 电阻 | 阻值、精度、封装、功率、环保等级 | 工程 + 品质 |
| CAP 电容 | 容值、耐压、材质、封装、精度、环保等级 | 工程 + 品质 |
| IC 芯片 | 品牌、MPN、封装、功能类别、温度等级、环保等级 | 工程 + 品质 |
| CON 连接器 | Pitch、Pin 数、高度、方向、镀层、品牌/MPN | 工程 |
| PI/FPC 材料 | 材料类型、厚度、铜厚、铜箔类型、胶系、尺寸、环保等级 | 工程 + 品质 |
| AUX 辅材 | 材料名称、规格、厚度、宽度、颜色、用途、环保等级 | 工程 + 仓库 |
| PCK 包装 | 类型、尺寸、客户专用标识、环保要求 | 仓库 + 品质 |

## 6. 建议的确认方式

1. 先确认 B01 至 B06，允许阶段 1 数据底座开始。
2. 用 50 至 200 条真实、已人工判定的物料样本确认 B07 至 B12。
3. 用一个真实产品 BOM 和一组客户专用/替代料场景确认 B13 至 B18。
4. 完成只读数据盘点后确认 B19 至 B24。
5. 每项决策记录确认人、日期、适用范围和后续变更流程，不只保留聊天结论。

## 7. 当前推荐默认值

在业务负责人正式确认前，实施计划只使用以下推荐作为评审基线，不写入生产代码：

- 在线 Site 为唯一生产主系统。
- 正式编码采用 `CYD-品类-6位流水号`。
- 自动匹配只作为建议，不自动生效。
- 新物料必须完成工程、品质、仓库和数据管理员审核。
- 首期范围内冲突必须清零。
- 替代和客户专用限制同时在 BOM、采购和领料环节执行。
- 迁移采用新增表、交叉映射和分模块切换，不直接覆盖旧数据。
