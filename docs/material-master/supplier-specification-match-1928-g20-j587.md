## 用户确认的业务语义

内部物料匹配以规格为主，不以供应商物料名称相似度为主。同一内部物料在不同
供应商文件中可以有不同名称；只有规格能够唯一对应现有内部物料时，清洗结果
才应给出内部编号。

规格不完整、关键属性冲突或多个内部编号同分时，系统不得任选编号。

## 解析修正

- 集中别名增加 `HC_CODE` → material code。
- 集中别名增加 `VendorCode` → supplier part number。
- 只有 `Description` 时，它以 `SUGGESTED` 身份进入 raw specification，
  同时可形成待审核名称候选，不自动确认。
- “描述”和“备注”同时存在时优先正式描述列；备注不再造成 description
  映射冲突。
- 标题/分段行只有在全部 Canonical 映射字段均为空时才标记
  `UNMAPPED_NON_DATA`；原始行仍保存，不按稀疏宽度误删真实物料行。

## 规格编号匹配

匹配不再使用名称相似度加分。结构化规格字段包括：

- 容量/阻值
- 误差
- 电压
- 封装
- 品类
- 已提供且双方存在的 MPN

规则：

1. 来源已提供的任一结构化规格与候选不一致，候选立即排除。
2. 候选自身定义的全部结构化规格均被来源覆盖并一致时，视为完整唯一规格。
3. 完整规格唯一一致才自动确认内部编号并标记自动匹配。
4. 部分规格只能给出唯一候选时可显示候选编号，但保持疑似并等待人工确认。
5. 多个候选同分时返回疑似匹配但不返回编号，防止随机选码。
6. 已确认的供应商料号映射仍具有最高优先级。
7. 电气量做语义归一，例如 `0.1uF = 100nF`、`5.0V = 5V`，
   `+5% = 5%`。
