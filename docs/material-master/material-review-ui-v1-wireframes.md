## 2. 审核工作台布局比较

### 2.1 方案 A：左侧完整内容 + 右侧 sticky 审核栏

```text
┌────────────────────────── 顶部物料摘要 ─────────────────────────────────────┐
│ 普通 FR4 覆铜板  [待审核]  V5  人工  PCB / 基材 / FR4 / 普通 FR4          │
├──────────────────────────────────────────────────┬───────────────────────────┤
│ 基础字段                                         │ 当前 Validation            │
│ 职责信息                                         │ 错误 0 / 警告 1            │
│ 类型化属性                                       ├───────────────────────────┤
│ 最近版本                                         │ 职责分离                    │
│ 最近变更                                         │ 当前用户可审核              │
│                                                  ├───────────────────────────┤
│                                                  │ 审核意见                    │
│                                                  │ [_______________________]  │
│                                                  │ [驳回修改] [审核通过]       │
└──────────────────────────────────────────────────┴───────────────────────────┘
```

优点：

- 属性增长时主栏自然向下扩展。
- Validation、职责限制和动作持续可见。
- 适合 1366×768 桌面连续审核。

结论：**批准采用并推荐**。

### 2.2 方案 B：顶部摘要 + 页签 + 底部操作区

```text
┌────────────────────────── 顶部物料摘要 ─────────────────────────────────────┐

├──────────────────────────────────────────────────────────────────────────────┤
│ 当前页签内容                                                                 │
│                                                                              │
│                                                                              │
├──────────────────────── sticky 操作区 ───────────────────────────────────────┤
│ 审核意见 [________________________]             [驳回修改] [审核通过]        │
└──────────────────────────────────────────────────────────────────────────────┘

优点：单屏结构规整。

问题：Validation、职责和属性被页签分散，审核前需要频繁切换，较难持续对照。

结论：保留为对比，不推荐。

│ 基础信息                                                     │
│ 职责信息                                                     │
│ 类型化属性（两列，进一步变窄时单列）                         │
│ 最近历史                                                     │
├──────────────── sticky 操作区 ───────────────────────────────┤
│ [驳回修改] [审核通过]                                        │
└──────────────────────────────────────────────────────────────┘
```

- 右侧栏移动到主体顶部并可折叠。
- 不继续压缩只读内容。
- 键盘顺序为摘要、Validation、职责、基础信息、属性、历史、审核意见、动作。

## 18. 1366×768 人工验收标记

1. 约 210px 左侧导航保留。
2. 队列筛选区不超过约 104px，首屏至少可见 8 条记录。
3. 队列 Validation 摘要在正常行高下仍可读。
4. 工作台左侧主栏可读，右栏约 280–320px。
5. 属性增加到 24 项以上时滚动稳定。
6. sticky 审核栏不覆盖正文或焦点环。
7. ERROR/WARNING 定位后目标属性和焦点环可见。
8. 驳回原因、字符计数和字段错误不被遮挡。
9. 对话框关闭后焦点回到触发位置。
10. 审核完成后队列筛选、排序、分页和成功反馈恢复。
11. 当前页最后一项被处理后只回退到最后有效页，不循环请求。

## 19. 禁止出现的元素

- 草稿编辑、创建或“审核并编辑”。
- 批量审核、多级审核、认领、转交或强制审核。
- Excel/CSV 导入、AI 分类或候选匹配。
- `submitted_by` 筛选控件，或把 creator 标成提交人。
- 前端加载全部待审数据后自行筛选。
- 自动驳回 Validation ERROR。
- 管理员自审例外。
- 自动版本覆盖、自动合并或结果未知时执行相反动作。
- 把意见、Key、CSRF、Validation 摘要或敏感正文写入 URL 或浏览器持久存储。
- 仅靠颜色、图标或动画表达 ERROR、WARNING、职责分离、加载或操作状态。
