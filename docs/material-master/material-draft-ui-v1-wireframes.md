## 2. 布局方案比较

### 2.1 方案 A：左分类与基本信息、右动态属性

```text
┌──────────────────────────────────┬───────────────────────────────────────────┐
│ 分类与基本信息                   │ 动态属性                                  │
│                                  │                                           │
│ 分类树                           │ 厚度 *          铜厚 *                    │
│ 标准名称 *                       │ TG *            阻燃等级 *                │
│ 基本单位 *                       │ 品牌 *          型号 *                    │
│ 采购 / 库存类型                  │ 无卤            颜色                      │
│ 品牌 / 制造商                    │ …                                         │
└──────────────────────────────────┴───────────────────────────────────────────┘
```

优点：短模板时两类信息同时可见，对照快速。

问题：属性较多时右栏成为长滚动区，左右阅读和错误定位分裂；1366px 内容区内两栏宽度容易不足。

结论：不采用。

### 2.2 方案 B：单列分区表单

```text
┌──────────────┬───────────────────────────────────────────────────────────────┐
│ 页面目录     │ 1. 分类                                                       │
│ 分类         ├───────────────────────────────────────────────────────────────┤
│ 基本信息     │ 2. 基本信息                                                   │
│ 动态属性     ├───────────────────────────────────────────────────────────────┤
│ 校验结果     │ 3. 动态属性                                                   │
│              ├───────────────────────────────────────────────────────────────┤
│              │ 4. 校验结果                                                   │
└──────────────┴───────────────────────────────────────────────────────────────┘
```

优点：阅读顺序稳定，复杂模板可持续向下扩展。

问题：分类和基础字段不能充分利用首屏宽度，首屏信息密度略低。

结论：作为窄宽降级参考，不作为桌面首选。

### 2.3 方案 C：混合分区 + 右侧快速定位

```text
┌──────────────────────────────────────────────────────────┬───────────────────┐
│ ┌──────────────────┬───────────────────────────────────┐ │ 快速定位          │
│ │ 分类             │ 基本信息                          │ │ 分类              │
│ └──────────────────┴───────────────────────────────────┘ │ 基本信息          │
│ ┌──────────────────────────────────────────────────────┐ │ 动态属性 1–12     │
│ │ 动态属性 1–12（全宽三列）                           │ │ 动态属性 13–24    │
│ └──────────────────────────────────────────────────────┘ │ 校验结果          │
│ ┌──────────────────────────────────────────────────────┐ ├───────────────────┤
│ │ 动态属性 13–24（如有）                              │ │ 服务端校验        │
│ └──────────────────────────────────────────────────────┘ │ 1 错误 / 2 警告   │
└──────────────────────────────────────────────────────────┴───────────────────┘
```

优点：分类与基础字段高密度并列，动态属性获得全宽，快速定位和 Validation 始终易发现；适合属性从少量增长到数十项。

结论：**批准采用**。

```text
│ sticky 操作区                                                │
└──────────────────────────────────────────────────────────────┘
```

- 右侧辅助栏移动到顶部并可折叠。
- 不继续压缩主体表单。
- 键盘顺序按分类、基本信息、动态属性、Validation、操作区排列。

## 19. 1366×768 人工验收标记

线框实施时必须人工确认：

1. 210px 左侧导航保留。
2. 主体宽度目标不少于约 860px。
3. 快速定位约 200px，不挤压主要控件。
4. 动态属性三列时 label、code、单位和 issue 均可读。
5. 属性增加到 24 项以上时页面滚动稳定。
6. sticky 底部操作区不覆盖最后字段。
7. 错误摘要定位后目标控件和焦点环不被 sticky 区域遮挡。
8. 对话框关闭后焦点回到触发位置。
9. 水平和垂直滚动不会产生双重不可控滚动区。

## 20. 禁止出现的元素

- 审核队列、批准或驳回按钮。
- Excel/CSV 导入、AI 分类或候选匹配入口。
- 正式物料修改、冻结、停用、替代或删除。
- source_type 非 MANUAL 选择器。
- source_ref 编辑控件。
- 自动合并、强制覆盖或版本恢复按钮。
- 用最近 5 条历史冒充可靠 `last_rejection`。
- 仅靠颜色、图标或动画表达 ERROR、WARNING、状态或加载。
- 把完整表单、Key、Token 或敏感正文放入 URL 或浏览器持久存储。
