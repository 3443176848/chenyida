# 通用规格来源识别与无序参数匹配

任务：`PHASE3-MATERIAL-LIBRARY-GENERAL-SPEC-MATCH-01`

状态：`DONE / DEPLOYED TO DEVELOPMENT SERVER`

日期：2026-07-18（Asia/Shanghai）

## 目标

不同物料品类的详细规格可能位于“规格、描述、品名及规格、物料名称”等不同
列中，参数顺序也不稳定。本任务只解决两个边界明确的问题：

1. 在已经识别的表格结构和字段 Mapping 中，确定哪一列实际承载详细规格。
2. 把来源规格和内部物料规格转换为带类型的参数集合，按集合比较相似度，
   不按原始字符串位置或排列顺序比较。

原始文件、不可变 Raw Row、完整原始规格和来源列继续保留；系统不调用 AI
猜测或补造原表中没有的参数。

## 规格来源选择

Parser 分别评估已映射规格列、多列组合、描述列和物料名称列。识别依据是可
确定提取的参数类型数和参数数；详细参数更多的来源优先，同等证据下优先明确
规格 Mapping。结果保存：

- 完整 `raw_spec`
- 独立 `raw_model/raw_mpn`
- `specification_source`
- 规格来源置信度和 Mapping 状态

只有型号而没有可确认详细规格时，型号只能作为人工复核候选，不能静默当成
通用规格。

## 参数模型与匹配

V1 确定性识别品类、封装、容量、阻值、电感值、电流、电压、功率、频率、
百分比误差、带单位绝对误差、介质/材质和尺寸，并统一量纲。参数按
`kind + normalized value` 比较，因此：

- 相同参数改变前后顺序，相似度不变。
- `0.1uF` 与 `100nF` 等确定性等价值相同。
- 同一类型双方有值但不一致时记录硬冲突并排除候选。
- 一侧缺少参数时保存缺项证据并降低置信度，不假装完整匹配。
- 完整参数集合只有在内部库唯一一致时才允许自动匹配。

MPN/型号和品牌作为独立标识证据展示，不混入通用规格相似度；MPN 相同不能
代替缺失的详细规格，MPN 不同也不改变相同通用规格参数本身的得分。已确认的
供应商料号映射继续按既有规则具有权威性。

## 数据与审核

本地迁移 `0004_cleaning_general_spec_tokens` 只扩展既有 `cleaning_rows`：

- `specification_source`

审核页面同时显示型号/MPN、完整原始详细规格、规格来源、来源参数和候选内部
参数。人工建档时 `value_spec` 保存已确认的完整详细规格，结构化字段作为附加
投影，避免电流、绝对误差等暂时没有独立列的参数丢失。旧 Cleaning Rows 不
回填、不重算；重新导入后才生成通用参数和新匹配证据。
