# 1928C 分项规格匹配

任务：`PHASE3-MATERIAL-LIBRARY-STRUCTURED-SPEC-MATCH-01`

状态：`DONE / DEPLOYED TO DEVELOPMENT SERVER`

日期：2026-07-18（Asia/Shanghai）

## 问题

1928C 的“物料型号”和“物料描述”是两个不同来源列。例如原始第 9 行的
规格证据分布为：

- 物料型号：独立厂商型号
- 物料描述：`CAP / 0201 / ±5% / NPO / 50V / 10PF / 型号 / 品牌`
- 生产厂家：独立列

旧逻辑只把物料型号作为 `raw_spec`，没有使用描述中的容量、误差、介质和
耐压，因此无法按真实规格匹配；若某个属性未提取，还可能让不完整候选获得
过高分数。

## 决定

匹配机制不得比较压缩后的整段规格文字，也不得比较名称相似度。系统分别从
原始规格、型号、描述和 MPN 中确定性提取以下属性：

1. 品类
2. 封装
3. 容量或阻值
4. 耐压
5. 误差
6. 材质或介质
7. MPN

每一项单独归一和比较。来源已经提供的容量、阻值、封装、耐压或误差只要有
一项与候选冲突，该候选立即排除。材质双方均有值时也必须一致；来源有材质而
内部物料未维护时，只能给出疑似候选，不得自动确认。

MPN 和已确认的供应商映射继续作为确定性证据，但不得通过猜解厂商料号字符
静默补造其他规格。

## 数据模型

新增本地版本化迁移 `0003_cleaning_structured_specification`，扩展现有
`cleaning_rows`，不建立第二套导入表：

- `raw_model`
- `raw_category`
- `parsed_tolerance`
- `parsed_material`

既有 `parsed_category`、`parsed_package`、`parsed_value`、
`parsed_voltage`、`raw_brand`、`raw_mpn` 和不可变 Raw Rows 继续复用。
旧清洗数据不回填、不重算；重新导入才使用新机制。

## 1928C 回归

截图对应行被分别提取为：

| 属性 | 结果 |
| --- | --- |
| 品类 | `CAP` |
| 封装 | `0201` |
| 误差 | `5%` |
| 材质/介质 | `C0G/NP0` |
| 耐压 | `50V` |
| 容量 | `10PF` |
| MPN | 从物料型号列独立保留 |
| 品牌 | 从生产厂家列保留 |

同时支持供应商常见确定性写法：

- `NPO`、`NP0`、`COG`、`C0G` → `C0G/NP0`
- `100P` → `100PF`
- `0.1uF` → `100nF`
- `+/-5%`、`±5%` → `5%`

当前内部测试物料没有 `10PF / 50V / 5% / 0201 / C0G/NP0`，所以截图行应
保持新物料，不能假装匹配已有编号。建立并审核对应内部物料后，同规格的不同
名称才能稳定返回该编号。

## 验证边界

- 规格分项专项覆盖截图行、单项冲突、容量简写和既有 1～5 匹配。
- 三份真实文件继续在隔离数据库导入，不写开发服务器正式数据。
- 本地迁移覆盖空库、已有数据、重复执行和失败回滚。
- 部署不清空、不重算当前 Cleaning Rows。
