# Material Import Workspace UI V1 — 低保真线框

> Status: APPROVED / IMPLEMENTED（非生产）
> 规格任务：PHASE2-TASK05；实施任务：PHASE2-TASK08
> 线框定义信息层级与交互语义；`PHASE2-TASK08` 已按此实现，Catalog 与性能/可访问性门禁均已通过。

## 1. 共用框架与标记

```text
┌ MaterialShell ─────────────────────────────────────────────────────────────┐
│ 面包屑 / 页面标题                                      [帮助] [用户]       │
├────────────────────────────────────────────────────────────────────────────┤
│ 状态/错误摘要（按需，纯文本，request_id 仅在服务端提供时显示）            │
├────────────────────────────────────────────────────────────────────────────┤
│ 主内容                                                                     │
└────────────────────────────────────────────────────────────────────────────┘
```

标记：`[ ]` 按钮，`(锁)` 尚不可访问，`(只读)` 可查看不可写，`✓` 已完成，`!` 失败/协调，`↔` 横向滚动。Stepper 状态用文字、图标和 `aria-current` 表达，不只靠颜色。

工作区共用头：

```text
批次 IMP-……  [批次状态]                 最近更新时间：……  [刷新] [取消*]
文件 ✓ ── 解析 ● ── Sheet与表头 (锁) ── 字段Mapping (锁) ── 确认 (锁)
```

`*` 只在服务端允许状态、用户有 capability、无冲突写/RESULT_UNKNOWN 且数据最新时出现。sticky 操作区不得遮住末项、焦点或横向滚动条。

## 2. 批次列表

```text
物料导入                                      [新建导入*]
状态 [全部⌄] 来源 [全部⌄] [✓]仅我创建  排序 [最新更新⌄]  每批 [20⌄]
────────────────────────────────────────────────────────────────────────────
批次号       状态              来源     创建时间        最近更新       操作
IMP-001      等待字段映射      XLSX     2026-…          2026-…        [查看]
IMP-002      字段映射已确认    CSV      2026-…          2026-…        [查看]
────────────────────────────────────────────────────────────────────────────
显示服务端本批结果                                  [下一批结果 →]
```

cursor 不显示、不记录、不解析。无 next_cursor 时按钮禁用；没有页码、总页数、上一 cursor 或无限追加。长批次安全摘要截断，但真实按钮可展开有界纯文本。

窄屏：筛选纵向排列，结果保持语义化表格并横向滚动；新建入口受 create capability 控制。

## 3. 新建导入与文件选择

```text
新建物料导入
1 选择文件   2 客户端预检与SHA   3 确认并创建/上传

文件（仅一个 .xlsx、.xls 或 .csv，不超过 10 MiB）
[选择文件]  尚未选择

说明：客户端检查不能代替服务端存储、安全和内容检查。
                                                        [取消] [继续（锁）]
```

文件控件有可见 label。文件名按纯文本安全清理；不预览或执行内容。焦点顺序：标题→文件控件→说明→取消→继续。

## 4. 客户端哈希

```text
已选择：超长文件名……xlsx   8.4 MiB   [重新选择]
客户端预检通过
正在计算 SHA-256：已处理 4.2 / 8.4 MiB
[真实字节进度条]                                      [停止计算]

文件尚未上传；此结果不表示文件安全或服务端会接受。
```

Worker 可终止，live region 克制播报阶段而非每个分块。重选使旧 Worker、SHA、重复确认和上传准备全部失效。

## 5. 用户确认、创建与文件上传

```text
确认上传
文件：……xlsx（8.4 MiB）
类型：浏览器报告的 MIME / 扩展名
SHA-256：已计算（不展示为访问凭证）
重复文件策略：默认拒绝
[返回重选]                              [确认创建批次并上传]

发送中：42%  （仅表示浏览器向网络发送字节）
[取消上传]
```

创建和上传分别显示操作状态。lengthComputable=false 时改为“正在上传”；100% 后改为“文件已发送，正在等待服务端存储和安全检查”。写已发送后对话框/页面不能假装取消未发生。

## 6. 重复文件确认

```text
┌ 检测到内容相同的历史导入文件 ───────────────────────┐
│ 原批次已失败：重复文件，上传未完成。                 │
│ 未创建解析结果，未复用其他批次，未创建Material Draft。│
│                                                     │
│ 若继续，将校验相同文件SHA并创建一个新的重试批次；   │
│ 原失败批次不会被修改。                               │
│                    [返回] [允许重复并创建新批次]     │
└─────────────────────────────────────────────────────┘
```

不显示其他批次号、创建人、文件名、时间、部门或对象信息。初始焦点在标题/返回安全动作；Tab 圈定；发送前 Escape 可关；发送后 busy、防重复并按结果恢复。

## 7. FILE_READY

```text
文件 ✓ ── 解析 ● ── Sheet与表头 (锁) ── Mapping (锁) ── 确认 (锁)

文件已就绪
服务端文件安全检查：基础检查通过
文件类型/大小/安全摘要（仅真实公开字段）

[取消批次*]                                      [启动解析*]
```

启动前仍会重新 GET；安全状态、parse capability、fresh 数据、无 pending/unknown 同时满足才启用。

## 11. PARSED 且 Mapping 准备中

```text
文件 ✓ ── 解析 ✓ ── Sheet与表头 ●(只读) ── Mapping (锁) ── 确认 (锁)

解析结果已发布，正在准备字段映射
准备状态：NOT_STARTED / QUEUED / RUNNING
[Sheet列表与Rows只读浏览]                          [手动刷新]
```

准备 FAILED 时改为“解析结果已发布，但字段映射准备失败”，提供刷新/联系管理员，无重新 parse 或伪重试。READY 是短暂态，只重读/有界轮询。

## 20. FAILED

```text
! 导入批次失败
阶段：文件存储 / 文件安全检查 / Parser / 资源限制 / 其他安全失败
安全说明：服务端允许的 message
request_id：……
[刷新] [返回批次列表] [允许的恢复动作*]
```

只按安全 failure_stage/code 分类；不显示对象、XML 路径、堆栈、SQL、Token/Cookie。Mapping preparation FAILED 使用第 11 节文案，不归为 parser FAILED。

## 21. CANCELLED

```text
批次已取消
该批次不能继续上传、解析或编辑Mapping。

[刷新] [返回批次列表]
```

## 22. VERSION_CONFLICT

```text
┌ 数据已发生变化 ──────────────────────────────────────┐
│ 服务端版本已更新。未保存的本地Mapping仍被保留，     │
│ 但不会自动覆盖最新数据。                             │
│                         [留在当前页] [刷新并核对]    │
└─────────────────────────────────────────────────────┘
```

刷新动作若会清 dirty，必须二次明确；不生成新 Key 自动重试旧载荷。

## 23. RESULT_UNKNOWN

```text
! 操作结果尚未确认
操作：上传 / 解析 / 取消 / 保存 / 预览 / 确认
写请求可能已被服务端接收。依赖操作已锁定。
[读取最新状态（辅助）] [使用原操作标识安全恢复]
request_id：……（若已有）
```

不显示“重试”造成新 Key 的歧义；恢复按钮必须复用原 endpoint、Key、不可变载荷。confirm unknown 时 Mapping 也不可编辑。

## 25. 对话框与焦点清单

| 对话框 | 初始焦点 | Escape/发送前 | 发送后 | 关闭后焦点 |
| --- | --- | --- | --- | --- |
| 允许重复文件 | 安全返回或标题 | 可取消 | busy；结果/unknown 处置 | 原触发按钮 |
| 启动解析 | 标题/返回 | 可取消 | busy、防重复 | 启动解析按钮 |
| 取消批次 | 返回 | 可取消 | busy；不假装未发送 | 取消按钮/状态标题 |
| 清除 Mapping 编辑 | 保留编辑 | 可取消 | 无远程写 | 原 Sheet/Header 控件 |
| 最终确认 Mapping | 返回 | 可取消 | busy；unknown 锁编辑 | 确认按钮/状态标题 |
| 离开未保存内容 | 留在页面 | 可取消 | 无远程写 | 原导航触发点 |
| 长单元格详情 | 关闭按钮/标题 | 可关闭 | 不涉及写 | 展开按钮 |

所有 modal 使用焦点圈定，背景不可交互，错误/长 warning 采用有界纯文本并允许滚动，不扩张到遮住主要动作。
