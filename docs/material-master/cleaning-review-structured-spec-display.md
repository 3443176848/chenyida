# 清洗审核分项规格对照

任务：`PHASE3-MATERIAL-LIBRARY-REVIEW-SPEC-DISPLAY-01`

状态：`DONE / DEPLOYED TO DEVELOPMENT SERVER`

日期：2026-07-18（Asia/Shanghai）

## 用户问题

清洗审核原来只显示“原始规格”和候选名称。1928C 的物料型号可能被显示在
原始规格列，而候选名称包含通用电气参数，人工无法判断系统究竟比较了什么。

厂商型号/MPN 与规格不是同一种字段：

- 型号/MPN 是厂商对具体器件的标识。
- 规格是品类、封装、容量/阻值、耐压、误差、介质/材质等属性集合。

## 实现

清洗审核表新增两组只读对照：

### 来源分项规格

- 品类
- 封装
- 容量/阻值
- 耐压
- 误差
- 介质/材质
- 型号/MPN
- 品牌

### 候选内部规格

从页面已经加载的内部物料列表按候选编码读取同一组字段。内部物料未维护的
字段明确显示“未维护”，不以候选名称字符串代替。

原列重命名为“原始型号/规格”，用于来源追溯，不作为页面端业务判断。
匹配、编号和确认权限继续由服务端决定，浏览器只展示证据。

## Parser 语义修正

当“物料型号”只有厂商型号，而“物料描述”包含两个及以上确定性结构化规格
证据时：

- `raw_model` 保存型号。
- `raw_spec` 保存原始描述中的规格来源。
- 后续 Matcher 仍分别提取属性，不比较整段字符串相似度。

来源介质存在但内部候选未维护时，仍保持疑似；置信度上限调整为 `0.95`，
不再出现缺少规格覆盖却显示 `1.0` 的情况。
