# 规格匹配精度门禁与扩展参数识别

任务：`PHASE3-MATERIAL-LIBRARY-SPEC-PRECISION-GATE-01`

状态：`DONE / DEPLOYED TO DEVELOPMENT SERVER`

日期：2026-07-19（Asia/Shanghai）

## 目标与准确性边界

本任务把“规格匹配必须精准”转化为可测试的确定性契约：

1. 已类型化且证据充分的规格，按归一化参数集合匹配，参数顺序不影响结果。
2. 同类型参数冲突时排除候选。
3. 规格证据不足时返回“规格不足”，不提供候选编号、不使用名称或品类补分。
4. 内部候选缺少来源已提供的参数时只能疑似，并向人工明确显示缺项。
5. 未识别的新语法保留完整原始规格，进入人工复核，不由 AI 猜测或补造。

任意未来供应商可能出现系统尚未定义语义的新写法，因此不能诚实承诺未知输入
永远 100% 自动识别。系统保证的是：已支持语法按确定性规则处理；不确定时
fail closed，不用高置信度掩盖信息不足。新增真实反例必须先加入回归夹具，再
扩展 Parser/Matcher。

## 规格证据门禁

`CATEGORY` 只用于类型约束，不作为足以区分内部编号的鉴别参数。

- 少于 2 类鉴别参数：`规格不足`，置信度 0，不返回候选。
- 至少 2 类鉴别参数：可参与候选比较，但缺项或非完整签名只能进入人工审核。
- 自动匹配：来源和内部候选都至少包含 3 类鉴别参数，且包含容量、阻值、电感、
  阻抗频率组合、带宽、尺寸、针数、间距或铜厚等锚点；参数集合完整一致、无
  冲突且候选唯一。
- 已人工确认的供应商料号映射继续作为独立权威证据，不受通用规格门禁替代。

Matcher 同时保存原始集合相似度、来源证据强度、校准后置信度、来源/候选缺项、
冲突和歧义候选数。审核页面显示“规格不足”和“候选内部规格缺少”的明确提示。

## 本轮新增确定性参数

- 分数功率：`1/20W`、`1/16W`
- 电压、电流、功率和频率范围
- 频率/阻抗组合，例如 `100MHz@90Ω`
- 带宽与 dB 参数
- 电阻嵌入写法，例如 `4K7`
- 长度/厚度、尺寸、针数、间距和铜厚
- Type-C/USB、FPC、SIM、TF、RF、板对板等接口语义
- IC、麦克风、滤波器、开关和天线品类

短写 `P` 只在有明确上下文时解释：电容上下文可作为 pF 简写；连接器/接口
上下文可作为针数。这样避免把 Type-C 描述中的 `16P` 错当成 `16pF`。

## 规格来源规则

规格 Mapping 不再把普通“型号”列直接拼入通用规格。除明确标题别名外，还对
每列样本值做规格丰富度统计，从未知标题中选择包含多类确定性参数的列。描述
或名称被选中时仍保留完整原文和来源列；只有型号、或显式规格不足两类鉴别
参数时，状态为 `SUGGESTED/NEEDS_REVIEW`，不得自动确认。

三份用户样本只在隔离数据库进行回归，附件和业务正文不提交：

- 1928C：25 条，24 条有可确认详细规格；1 条只有型号，不再冒充规格。

- J587：122 条均保留规格来源。隔离复算为 105 条新物料、5 条疑似、12 条规格
  不足；旧逻辑中 4 条只凭连接器大类得到的错误候选全部消失。
- J587 的 5 条疑似均为电容完整参数接近内部编号 2，但来源有介质、内部候选
  未维护介质，因此保持人工审核并显示内部缺项，不自动确认。

`1/20W` 和 `1/16W` 分别归一为 `0.05 W` 和 `0.0625 W`；Type-C `16P` 归一为
16 针，不生成电容值。
