# 物料主数据治理落地包

用途：解决物料数量大、供应商命名不统一、人手不足导致的录入和维护困难。

执行目标：

- 建立晨亿达内部标准物料库，BOM、采购、库存、生产、成本统一使用内部物料编码。
- 保留供应商原始名称和供应商料号，但只作为映射关系，不直接进入 BOM 主数据。
- 用批量导入、自动解析、相似匹配、人工审核的方式减少手工录入。
- 先治理高频料、核心材料、客户专用料和新增物料，不一次性清洗全部历史数据。

文件说明：

| 文件 | 用途 |
| --- | --- |
| `01-编码规则.md` | 内部物料编码、品类编码、命名规则 |
| `02-字段字典.md` | 标准物料、供应商映射、待清洗池等字段定义 |
| `03-导入清洗流程.md` | 从供应商 Excel 到内部物料库的处理流程 |
| `04-岗位分工与审核SOP.md` | 采购、工程、品质、仓库、财务的审核职责 |
| `templates/` | 可直接填写或导入系统的 CSV 模板 |
| `tools/material_cleaner.py` | 本地物料清洗样例工具，验证匹配规则 |

首周执行顺序：

1. 选 3 到 5 家核心供应商，收集报价表、送货单、历史采购表。
2. 用 `templates/供应商原始物料导入模板.csv` 统一整理原始数据。
3. 用 `templates/内部标准物料库.csv` 建一批高频内部标准物料。
4. 运行 `tools/material_cleaner.py`，生成待确认清洗结果。
5. 采购批量确认高置信度匹配，工程和品质处理疑似匹配和新物料。
6. 确认后的数据写入 `供应商物料映射表` 和 `内部标准物料库`。

最低上线标准：

- 新增物料必须先查重，再审核建档。
- 采购下单可以输入供应商料号，但系统必须能映射到内部物料编码。
- BOM 不允许直接使用供应商原始名称。
- 客户专用料、条件替代料必须有明显标识。
